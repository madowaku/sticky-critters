# 画像プレビューが出ない

これは会社PCで一番見たいです。
原因候補はだいたい3つです。

```txt
A. convertFileSrc は効いているが、asset protocol / CSP / 権限で読めていない
B. pathは入っているが Windows path の扱いがズレている
C. 会社PCの画像パスが OneDrive / ネットワーク / 権限付きフォルダで WebView から読めない
```

まず確認したいのは、画像付箋に保存されている `path` が本当にこういう形式かです。

```txt
C:\Users\...\Pictures\image.png
```

もし `file.name` だけ、または空なら、ドロップ経路がブラウザ fallback 側に落ちています。

Codexにはこう投げるのが良いです。

```txt
画像付箋のプレビューが会社PCのTauriアプリで表示されない問題を調査・修正してください。

確認してほしいこと:
1. image付箋の note.path が絶対パスになっているか
2. getImagePreviewSrc / convertFileSrc の結果を dev console に出す
3. img の onError で失敗理由を分かる範囲でログ表示
4. Windows path / OneDrive path / 日本語パス / スペース入りパスで表示できるか確認
5. asset protocol / CSP / capabilities が不足していないか確認

要件:
- pathを直接 img src に入れない
- Tauri環境では convertFileSrc を使う
- 失敗時は「プレビューできません」だけでなく、Open と Copy Path は使える状態にする
- ブラウザfallbackは壊さない
- 実ファイル削除やshell pluginは追加しない

完了条件:
- Tauriアプリでローカルpng/jpg/webpのプレビューが出る
- 失敗時にplaceholder表示
- npm run lint / npm run build / npm run tauri:build が通る
```

# 付箋をもう少し小さくしたい

これはかなり重要です。実用時に効きます。

今の課題は、カード上に操作ボタンが多すぎることですね。

```txt
Open はよく使う
Copy Path / Alarm / Today / Stash / Lock / etc は常時表示しなくていい
```

おすすめは **Compact Note Card** です。

## 方針

```txt
通常表示:
- タイトル/種類
- 本文プレビュー
- Open / Copy など主要1〜2個だけ
- 右上に「…」メニュー

「…」クリック:
- パスをコピー
- アラーム設定
- 今日だけ
- ほっぺに預ける
- ロック
- 色変更
- ヤギにあげる
```

設定に追加するなら：

```ts
noteActionMode: "expanded" | "compact"
```

まずはデフォルト compact でも良いです。

Codex指示：

```txt
付箋カードを少し小さく使えるように、カード上のアクションを整理してください。

目的:
常時表示ボタンが多くて付箋が大きくなるため、使用頻度の低い操作を「…」メニューに移動します。

方針:
- Open / Copy など主要操作は残す
- パスをコピー、アラーム設定、今日だけ、ほっぺに預ける、ロック、ヤギにあげる等は「…」メニューに移動
- locked時は禁止操作をdisabled
- 既存機能は削らない
- モバイルではなくWindowsデスクトップ前提で、クリックしやすさは維持

追加:
- Settings > Appearance に noteActionMode: compact / expanded を追加してもOK
- まずは compact をデフォルトにしてもOK

完了条件:
- 付箋の最小高さが下がる
- 主要操作は迷わない
- 既存操作は「…」から使える
- npm run build が通る
```

# mac対応はどう？

**技術的には可能、配布まで含めると少し面倒**です。

Tauri自体は macOS / Linux / Windows など複数デスクトップ向けにビルドできるフレームワークです。Tauri 2.0の公式ブログでも主要デスクトップOS向けに小さく高速なバイナリを作るフレームワークとして説明されています。([Tauri][1])

ただし、macOSで一般配布するにはコード署名やnotarizationが絡みます。Tauri公式のmacOS署名ドキュメントでも、notarizeにはAppleに認証するための認証情報が必要とされています。([Tauri][2]) また、配布全般としてコード署名が多くのプラットフォームで必要になる、とTauriの配布ドキュメントにもあります。([Tauri][3])

なのでおすすめは：

```txt
今すぐ:
Windows betaに集中

次:
GitHub Actionsで macOS build が通るかだけ確認

その後:
身内テスト用の unsigned .app / dmg

公開配布:
Apple Developer Program + signing + notarization を検討
```

さらにSticky Crittersは以下のOS依存機能が多いです。

```txt
トレイ
自動起動
通知
ホットキー
Click-through
Kangaroo Pocket
ローカルパス / folder open
```

なので、mac対応は **「移植はできるけど、各機能の確認が多い」** です。
v0.1 betaではWindows特化でOK。macは `v0.3 candidate` くらいで良いと思います。

# 2. 箇条書きの完了項目だけ×印で消す

これはめちゃくちゃ便利です。
ただし、いきなり構造化タスク管理にすると重くなるので、最初は **テキスト付箋内の簡易チェックリスト** が良いです。

## おすすめ仕様

本文にこう書いたら：

```txt
- 牛乳を買う
- メール返信
- npm run build
```

表示時に行ごとに認識して、各行に小さな `×` か `✓` を出す。

押すと、その行を「完了」扱いにする。

表示は例えば：

```txt
× 牛乳を買う
メール返信
npm run build
```

ただ、「×で消す」だと本当に削除なのか完了なのか少し迷うので、内部的には2案あります。

## A案: 行を本当に削除する

メリット：

* シンプル
* 付箋が短くなる
* 一時メモ向き

デメリット：

* 間違えて消すと戻しづらい
* 完了履歴が残らない

## B案: 完了として打ち消し線

メリット：

* 安心
* 達成感がある
* undoなしでも分かりやすい

デメリット：

* 付箋は少し残る

Sticky Crittersなら、まずは **B案: 打ち消し線** が良さそうです。
「×印で消す」見た目にしつつ、実際は行頭を `[x]` に変える。

```txt
- 牛乳を買う
```

を

```txt
- [x] 牛乳を買う
```

に変える。

または元から：

```txt
- [ ] 牛乳を買う
- [ ] メール返信
```

にも対応。

## 最小仕様

```txt
対応する行:
- item
- [ ] item
* item
・item

完了後:
- [x] item
```

表示時：

* `[x]` の行は打ち消し線
* 未完了行には小さな ✓ or × ボタン
* locked note では操作不可

## Codex向け指示案

```txt
テキスト付箋に簡易チェックリスト機能を追加してください。

目的:
一つの付箋に箇条書きしたタスクのうち、完了した項目だけをワンクリックで完了扱いにできるようにします。

対象:
- kind: plain の付箋
- locked note では操作不可
- code/url/file/folder/image/sketch は対象外

仕様:
1. plain付箋の body を行単位で解析する
2. 以下の行を checklist item として扱う
   - "- item"
   - "* item"
   - "・item"
   - "- [ ] item"
   - "- [x] item"
3. checklist item の未完了行には小さな ✓ または × ボタンを表示
4. 押すとその行を "- [x] item" に変換する
5. "- [x] item" は打ち消し線で表示する
6. 既に "- [x]" の行には「戻す」操作を付けてもOK。最小実装では不要
7. 変更時に body / updatedAt / lastInteractedAt を更新
8. notes.json に保存する
9. 通常の文章行は今まで通り表示
10. 編集機能と競合しないようにする

文言:
ja:
- checklist.done: 完了
- checklist.undo: 戻す

en:
- checklist.done: Done
- checklist.undo: Undo
```
